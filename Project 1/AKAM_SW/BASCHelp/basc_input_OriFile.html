(*.ori)

The orientation file contains data pertaining to each of the images; like the approximate values for exterior orientation parameters (EOP) of all involved images in addition to the ID of the camera used to take that image, and the image coordinate measurement accuracy, etc.

It should be in the following format:

  • Photo ID: photo ID. This ID should match the corresponding image ID in the image coordinates file (*.icf). It cannot contain any white space within its string (see example below).
  • Camera ID: the ID of camera used to capture this photo. This ID should be identical to one of the camera IDs in the camera file (*.cam). It cannot contain any white space within its string (see example below).
  • Sigma xy: image point measurement accuracy.
  • Time tag that corresponds to the moment of capture of the image in question. This value is very crucial for BASC adjustment procedures that involve GPS and/or INS constraints
  • EOP rotation angles: omega, phi and kappa for this image, always in degrees.
  • EOP perspective center coordinates: Xo, Yo, Zo of this image.
