The BASC program requires the use of ASCII files as the input media of needed data. These input files have specific structure in which data is arranged. It is crucial to exactly follow this structure to correctly read the file contents. BASC does not validate the structure of input files. Examples of such input files are presented after the description of each file. 

A comment line is any line that starts with the ‘!’ character. Comment lines are skipped in all input files. Such comments can be inserted on any line but not between data elements within a line. Supplemental comments in the files should not be added, and one should pay attention to the carriage returns.

Important:
Please note the following about the system of units used in BASC:

  • All angles are inputted and outputted in degrees. Variances of such angles are displayed in arc seconds.
  • Image coordinates can be measured using any unit of choice (mm or pixels) bearing in mind that the output for the principle distance, principle point coordinates (xp, yp, c) and distortion parameters will follow the chosen input units.
  • All object space linear information units should be consistent, meters for example. Such information includes the positional portion of the EOPs, GPS observations at the aircraft, ground coordinates of object points, and measured ground distances
Mandatory input files:
  • Project File
  • Image Coordinate File
  • Ground Control Points File
  • Camera File
  • Orientation File

Optional input files:

Based on the options entered within the Project File, one or more of the following input files might be required:

  • GPS/INS File
  • Distance File
